# Portfolio Website Plan

## 1) Proje Vizyonu
- Amaç: Teknoloji geliştiricisi + kurucu/girişimci kimliğini güçlü ve güven veren bir şekilde yansıtmak.
- Konumlandırma: "AI araçlarını etkin kullanan, fikirden ürüne hızlı çıkan builder-founder."
- Ana mesaj: Ürün düşünebilen, kodlayabilen, AI ile hızlanan ve teslim odaklı bir geliştirici.

## 2) Kişisel Profil (Brand Narrative)
- Rol: Teknoloji geliştiricisi, ürün odaklı kurucu adayı/girişimci.
- Yetkinlik: Yazılım geliştirme, GitHub iş akışları, ürün geliştirme süreci, teknik problem çözme.
- Çalışma tarzı:
  - AI-assisted development (Gemini, Antigravity, Codex, Claude, Claude Code).
  - Hızlı prototipleme -> iterasyon -> üretime alma.
  - Veriye ve kullanıcı geri bildirimine dayalı iyileştirme.

## 3) Hedef Kitle
- Potansiyel iş ortakları / kurucu partnerler
- Freelance veya ürün geliştirme müşterileri
- Teknik ekipler ve işe alım yapan şirketler
- Yatırımcı veya proje destekçileri (uzun vadede)

## 4) Site Hedefleri (KPI)
- İletişim formu dönüşümü
- GitHub / LinkedIn tıklamaları
- Proje sayfalarında geçirilen süre
- CV / profil indirme ya da görüntüleme oranı

## 5) Bilgi Mimarisi (Site Haritası)
- Home
- About
- Projects
- Tech Stack & Tools
- Process
- Contact

## 6) Bölüm Planı (Section by Section)

### 6.1 Hero (Ana Ekran)
- Başlık: Kurucu bakış açısına sahip AI-native geliştirici.
- Alt metin: Fikirden çalışan ürüne hızlı geçiş, modern araçlarla üretim.
- CTA:
  - "Projelerimi İncele"
  - "İletişime Geç"

### 6.2 Hakkımda (About)
- Kısa profesyonel hikaye (1-2 paragraf)
- Neyi çözdüğün: ürün, hız, kalite, teslim.
- Değer önerisi: teknik + ürün + AI çarpanı.

### 6.3 Projeler (Featured Projects)
- Her proje kartında:
  - Problem
  - Çözüm
  - Kullandığın teknoloji
  - Sonuç / etki (mümkünse metrik)
  - Linkler (Live, GitHub, Case Study)
- Projeleri "öne çıkanlar" + "diğer projeler" olarak ayır.

### 6.4 Tech Stack & AI Tools
- Diller/teknolojiler (ör. JS/TS, Python, vb.)
- Platformlar (GitHub, CI/CD vb.)
- AI araç seti:
  - Gemini
  - Antigravity
  - Codex
  - Claude
  - Claude Code
- Not: "Aracı değil çıktıyı optimize eden yaklaşım" vurgusu.

### 6.5 Çalışma Süreci (Process)
- 1. Keşif (problem + hedef)
- 2. Hızlı prototip (MVP)
- 3. Geliştirme + test
- 4. Yayınlama
- 5. Ölçümleme ve iterasyon

### 6.6 Güven Bölümü (Credibility)
- GitHub katkıları / öne çıkan repo bağlantıları
- Varsa referanslar, kısa yorumlar, başarılar
- Kısa "neden birlikte çalışmalı?" özeti

### 6.7 İletişim (Contact)
- E-posta
- LinkedIn
- GitHub
- Kısa form: İsim / E-posta / Proje fikri
- Net CTA: "Birlikte proje başlatalım."

## 7) Tasarım Prensipleri
- Modern, temiz, profesyonel ve güven veren görünüm
- Tipografi: teknik ama sıcak bir ton
- Karanlık/açık tema opsiyonu (opsiyonel)
- Mobil öncelikli responsive tasarım
- Hız ve erişilebilirlik (Lighthouse odağı)

## 8) İçerik Gereksinimleri (Toplanacaklar)
- Kısa bio (50-80 kelime)
- Uzun bio (150-250 kelime)
- Proje listesi (en az 3, ideal 5-6 proje)
- Her proje için:
  - Kısa özet
  - Stack
  - Görsel/screenshot
  - Linkler
- Profil foto veya kişisel görsel stil
- İletişim bilgileri + sosyal medya linkleri

## 9) Teknik Plan (Öneri)
- Frontend: Next.js + TypeScript
- UI: Tailwind CSS + component yapısı
- İçerik yönetimi:
  - Başlangıç: statik JSON/MDX
  - Sonra: headless CMS (isteğe bağlı)
- Deploy: Vercel
- Analytics: Plausible veya GA4

## 10) MVP Kapsamı (v1)
- Home + About + Projects + Contact
- En az 3 güçlü proje case’i
- Temel SEO (title, description, OG)
- Performans optimizasyonu (image + lazy load)

## 11) Sonraki Aşama (v2)
- Blog / build log
- Proje detay sayfaları (case study formatı)
- Çoklu dil (TR/EN)
- Mini ürün vitrin sayfası

## 12) Uygulama Sırası
1. İçerik toplama
2. Wireframe
3. Görsel tasarım
4. Geliştirme
5. Test + SEO
6. Yayınlama

## 13) Açık Kararlar
- Kişisel isim/domain netleşmesi
- Renk paleti ve görsel stil yönü
- İlk yayında TR mi EN mi yoksa iki dil mi
- Projelerin sıralama kriteri (etki, teknik zorluk, güncellik)
